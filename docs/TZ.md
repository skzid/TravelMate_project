# Техническое задание

## Название проекта

TravelMate — дневник путешествий.

## Цель проекта

Создать учебное веб-приложение, в котором пользователь может хранить заметки о посещённых или интересных городах, загружать фотографии и получать географические данные через внешний API.

## Пользователи

Основной пользователь — ученик или путешественник, который хочет сохранять небольшие заметки о городах и делиться публичными записями с другими.

## Основные функции

1. Регистрация нового пользователя.
2. Авторизация и выход из аккаунта.
3. Добавление записи о городе.
4. Редактирование и удаление своих записей.
5. Загрузка изображения к записи.
6. Автоматическое определение страны и координат города через API геокодера.
7. Разделение записей на публичные и личные.
8. Просмотр публичной ленты.
9. REST API для получения и создания публичных записей.

## Требования к данным

В базе данных должны храниться:

- пользователи: имя, почта, хеш пароля, описание, дата регистрации;
- записи: название, город, страна, координаты, описание, имя файла изображения, приватность, дата создания, автор.

## Требования к интерфейсу

Интерфейс должен быть простым и понятным. Для оформления используется Bootstrap. Страницы должны открываться в браузере и корректно отображать формы, ленту записей и описание API.

## Требования к технологиям

В проекте должны быть использованы:

- Flask;
- Jinja2-шаблоны;
- Flask-WTF;
- Flask-Login;
- SQLAlchemy ORM;
- SQLite;
- Bootstrap;
- загрузка файлов;
- внешний HTTP API;
- REST API;
- requirements.txt.

## Сценарии использования

1. Пользователь регистрируется и входит в аккаунт.
2. Пользователь нажимает «Добавить место».
3. Пользователь вводит название, город, описание, выбирает фотографию и приватность.
4. Приложение определяет страну и координаты.
5. Запись появляется в ленте.
6. Если запись публичная, её видят другие пользователи и REST API.
